# 启动循环和重启循环

# 刷新循环

刷新循环是由于安装了不兼容的插件导致的症状，使设备在越狱状态下陷入不断重新加载桌面的循环。

虽然很难诊断刷新循环的具体原因，但最可能的罪魁祸首是你最近安装/更新的插件与你的 iOS 版本或其他插件不兼容。

# 半连接/半完全越狱

TIP

半连接越狱是指重启后需要借助电脑上的应用来重新越狱。

半完全越狱是指重启后需要借助设备上安装的应用或网站来重新越狱。

要摆脱刷新循环，请强制重启你的设备。重启后，使用你选择的越狱工具，在越狱设置中禁用插件注入。成功越狱后，你应该能够访问你的包管理器，移除最近安装的插件或最近更改的可能导致刷新循环的插件设置。

在最坏的情况下，如果你无法诊断或修复问题，请在 r/jailbreak Discord 服务器在新窗口中打开上寻求帮助，或恢复根文件系统。

# 完美越狱

TIP

完美越狱是指重启后仍然保持越狱状态而无需重新越狱的状态。

完美越狱上的刷新循环很容易与启动循环混淆，因为即使重启设备，你仍然会处于刷新循环中。

要摆脱可能的刷新循环，你必须强制重启设备并按住音量加键，具体时机因越狱工具而异，最好的办法是按住音量加键直到进入 iOS。

即使你多次尝试，可能会发现按住音量加键也无法进入 iOS；此时你可能陷入了启动循环。

# 启动循环

启动循环是由系统文件损坏或硬件故障引起的不可恢复的错误。虽然在现代 iOS 版本中较为罕见，但仍有通过意外或故意手段导致设备启动循环的可能。

启动循环的特征是即使强制重启后，Apple 标志仍不断反复出现。

要解决启动循环，你必须对设备进行 DFU 恢复。

# 如何进入 DFU 模式

# 带有实体 Home 键的设备（iPhone 6s 及以下

  1. 同时按住 Home 键和锁屏键。
  2. 8 秒后，松开锁屏键，继续按住 Home 键。

# iPhone 7/7+、iPod 第 7 代和 2018 年款 iPad

  1. 如果设备尚未开机，请先开机
  2. 使用数据线将其连接到电脑
  3. 启动 iTunes（或 macOS Catalina 及更新版本上的 Finder），确认 iPhone/iPad 已被识别
  4. 同时按住侧边键和音量减键。
  5. 8 秒后，松开侧边键，继续按住音量减键。

# 无 Home 键的设备、iPhone 8/8+、iPad Air 5 和 iPad Mini 6

  1. 如果设备尚未开机，请先开机
  2. 使用数据线将其连接到电脑
  3. 启动 iTunes（或 macOS Catalina 及更新版本上的 Finder），确认 iPhone/iPad 已被识别
  4. 快速按下音量加键，然后立即按下音量减键
  5. 按住电源键直到屏幕变黑
  6. 松开电源键
  7. 立即同时按住电源键和音量减键 5 秒钟
  8. 松开电源键，继续按住音量减键再 5 秒钟

# 如何退出 DFU 模式

# 带有实体 Home 键的设备（iPhone 6s 及以下）

同时按住 Home 键和锁屏键，直到设备重启。

# iPhone 7/7+、iPod 第 7 代和 2018 年款 iPad

同时按住侧边键和音量减键，直到设备重启。

# 无 Home 键的设备、iPhone 8/8+、iPad Air 5 和 iPad Mini 6

  1. 快速按下音量加键
  2. 快速按下音量减键
  3. 按住侧边键，直到设备重启。

# Chimera、Odyssey 和 Taurine 的常见错误

# ERR_Jailbreak

DANGER

如果你在使用 taurine-permanent 时遇到此问题，请转而参考使用 taurine-permanent 时出现 ERR_Jailbreak。

这是由于你之前使用过其他越狱工具导致的。要解决此问题，你需要恢复根文件系统：

  1. 重启设备。
  2. 根据你的 iOS 版本打开 Odyssey 或 Taurine 应用。
  3. 开启 恢复根文件系统 选项
  4. 点击越狱按钮。

恢复根文件系统成功后，请再次尝试越狱。

# ERR_Already_Jailbroken 和 ERR_KernRW（Taurine）/ ERR_TFP0（Odyssey）

ERR_Already_Jailbroken 表示越狱过程被中断。ERR_KernRW 和 ERR_TFP0 表示漏洞利用失败。两者都可以用同样的方法解决：

  1. 重启设备。
  2. 根据你的 iOS 版本打开 Odyssey 或 Taurine 应用。
  3. 点击越狱按钮。

TIP

越狱很少能 100% 成功，你可能需要多次重新尝试运行越狱工具。

# 在 Odyssey 1.4.2 上首次越狱后打开 Sileo 时出现"Installation Error"

移除当前安装的 Odyssey 版本，侧载最新版本的 Odyssey在新窗口中打开，然后恢复根文件系统并重新越狱。

# 在 Taurine 上使用一段时间后无法打开 NewTerm 或 Cr4shed（以及其他应用）

TIP

如果你使用的是 14.3 及更高版本，或者拥有 A12 或更新设备，很可能是你重启了设备（这意味着你不再处于越狱状态），或者可能出现了与越狱完全无关的其他问题。

如果你使用的是 14.0 到 14.2 版本，并且拥有 A11 或更早设备，此问题是由于某些插件和应用使用的库中包含可能在越狱工具停止代码签名之前运行的组件。发生这种情况时，受影响的插件和应用将无法使用。

此问题最简单的解决方案是重启设备并重新越狱。如果你不希望出现此问题，可以使用 Odysseyra1n 越狱，因为 Taurine 和 Odysseyra1n 相互兼容，而且 Odysseyra1n 不受此问题影响。

# 使用 taurine-permanent 时出现 ERR_Jailbreak

更新到最新版本的 taurine-permanent。

# 在 Chimera 1.6.1 或 Taurine 1.1.2 上首次越狱后 Sileo 未出现

  • 如果你在 Chimera 上遇到此问题，请获取最新版本的 Chimera在新窗口中打开
  • 如果你在 Taurine 上遇到此问题，请获取最新版本的 Taurine在新窗口中打开

# unc0ver 的常见错误

# unc0ver 在 iOS 14 上的常见问题

TIP

虽然这些问题在 iOS 14 上最为常见，但部分问题也可能出现在更早的 iOS 版本上。

在 iOS 14 上使用 unc0ver 时，经常会遇到以下问题： - 尝试启动应用时卡住 - 各种与应用商店相关的问题（包括无法下载应用或进行应用内购买） - 越狱状态下无法接听电话 - 使用蜂窝数据时出现问题（例如无法加载网页或其他在线服务） - 无法保存、拍摄或查看截屏 - Safari 无法加载页面，或无限重新加载页面 - 分享菜单无法正常工作 - 连接 Carplay 时出现问题。

WARNING

如果你使用 Fugu14 对 unc0ver 进行永久签名，可能会更频繁地遇到这些问题，以及一些 Fugu14 特有的问题，例如： - 无法查看电池统计信息 - 更频繁和随机发生的内核崩溃

要临时解决这些问题，请在设备上的终端应用（例如 NewTerm）中输入 launchctl reboot userspace 进行用户空间重启。

# 根文件系统已挂载

  1. 打开"设置"应用
  2. 进入 通用 -> iPhone 存储空间
    • 可能显示为iPhone 存储空间、iPad 存储空间或iPod 存储空间
  3. 滚动到已下载的系统更新并点击它
  4. 点击删除
  5. 重新运行 unc0ver

如果这样无法解决，或者没有已下载的更新，你可能需要下载 OTA 更新后再删除。请尝试以下步骤：

  1. 打开 unc0ver 应用
  2. 点击右上角的设置图标
  3. 取消勾选所有选项
  4. 启用 恢复根文件系统
  5. 点击右上角的 Done
  6. 前往越狱选项卡，点击 恢复根文件系统
    • 如果按钮显示为 Jailbreak，请返回检查是否只启用了 恢复根文件系统 选项
  7. 当提示恢复时，点击 OK
    • 这可能需要一些时间
    • 如果失败，请从头重新尝试
  8. 完成后，再次点击 OK，设备将重启
  9. 打开"设置"应用
  10. 进入 通用 -> 软件更新
  11. 下载软件更新，但不要安装它
  12. 返回 通用
  13. 下载完成后，像之前一样进入 iPhone 存储空间
    • 可能显示为iPhone 存储空间、iPad 存储空间或iPod 存储空间
  14. 滚动到已下载的系统更新并点击它
  15. 点击删除
  16. 滚动到 unc0ver 应用并点击它
  17. 点击删除
  18. 像之前一样重新安装 unc0ver，但暂时不要运行它
  19. 打开飞行模式，然后运行越狱
    • 这可能需要多次尝试，请持续尝试直到成功

TIP

如果这不起作用，请在 r/Jailbreak Discord 服务器在新窗口中打开上寻求帮助。

# 使用 Batchomatic 或 IAmLazy 等插件后无法在 unc0ver 中重新越狱或恢复根文件系统

如果你陷入此状态，能否修复此问题取决于你的 iOS 版本：

  • 如果你使用的是 14.3 及以下版本的 iOS（不包括 11.x 或 A12(X) 12.1.3-12.4.1），请按照其他越狱工具的恢复根文件系统方法来操作，前提是该工具支持你的设备/版本组合。
  • 如果你使用的是 iOS 14.4 到 14.5.1，请尝试在 Fugu14 应用中恢复根文件系统。
  • 如果你使用的是 iOS 14.6 到 14.8，或者属于第一组中排除的设备，你唯一的选择是等待其他越狱工具、等待修复此问题的 unc0ver 更新，或者 DFU 恢复到最新的 iOS 版本并在相当长一段时间内失去越狱能力。

# 修复 Sileo 上的软件包安装错误

在本故障排除部分，我们假设你在 Taurine/Odyssey/Odysseyra1n 上运行 Sileo。

要确定你遇到的错误类型，请在 Sileo 中安装或移除软件包遇到错误时点击显示详情。

如果以下修复方法没有帮助，请随时在 Sileo Discord 服务器在新窗口中打开上寻求帮助。

# 语法错误："(" 意外的符号

此错误表示你要安装/移除的插件开发者在编写安装/移除脚本时出现了错误。你需要使用 Filza 来解决此问题。

# 移除导致此错误的插件

  1. 打开 Filza 并导航到 /Library/dpkg/info
  2. 找到与出错插件对应的 .postinst 或 .postrm 文件，例如如果 ShortMoji 2 出现安装错误，需要编辑的文件是 com.miro.shortmoji2.postinst；如果是卸载错误，则是 com.miro.shortmoji2.postrm
  3. 长按该文件直到出现菜单
  4. 点击打开方式
  5. 点击文本编辑器
  6. 找到文件第一行内容 #!/bin/sh
  7. 将 sh 改为 bash
  8. 点击保存
  9. 打开 Sileo
  10. 卸载该插件
    • 对于付费插件，如果你希望安装该插件，需要向开发者提交错误报告
    • 对于免费插件，你应该向开发者提交错误报告。如果你希望立即安装该插件，请继续按照本指南操作

# 编辑插件以修复错误（仅限免费插件）

  1. 在设备的 Safari 中打开 Parcility在新窗口中打开
  2. 搜索你要安装的插件
    • 如果搜索结果中未出现，你需要等待开发者更新他们的插件
  3. 点击下载
  4. 文件下载完成后，在 Safari 的下载菜单中点击该文件
  5. 点击分享按钮
  6. 点击保存到 Filza
  7. 点击保存
  8. 打开 Filza，它将打开到你下载插件文件的文件夹
  9. 点击你要修复的插件
  10. 向下滚动并点击解压
  11. 打开结果文件夹，并导航到 DEBIAN
  12. 重复上面"移除导致此错误的插件"中的步骤 2-8
  13. 导航回下载插件的文件夹
  14. 长按插件文件夹
  15. 点击创建 DEB
  16. 长按生成的 deb 文件
  17. 点击打开方式
  18. 点击 Sileo
  19. 点击获取

TIP

越狱社区的一些开发者在他们的安装/移除脚本中调用了错误的 shell。在 2021 年 4 月之前，大多数越狱工具将 /bin/sh 指向 /bin/bash，因此开发者即使使用了 Bash 特性，也可以使用 #!/bin/sh 来指定使用的 shell。2021 年 4 月，运行在 Chimera、Odyssey、Taurine 和 Odysseyra1n 上的 Procursus 切换到了另一个性能更高的 shell，这导致了这些格式不正确的脚本出现问题。

# 执行格式错误

此错误表示你要安装/移除的插件开发者在安装/移除脚本中遗漏了关键部分。你需要使用 Filza 来解决此问题。

# 移除导致此错误的插件

  1. 打开 Filza 并导航到 /Library/dpkg/info
  2. 找到与出错插件对应的 .postinst 或 .postrm 文件，例如若 ShortMoji 2 出现安装错误，需要编辑 com.miro.shortmoji2.postinst
  3. 长按该文件直到出现菜单
  4. 点击打开方式
  5. 点击文本编辑器
  6. 在文件开头添加一行新内容：#!/bin/bash
  7. 点击保存
  8. 打开 Sileo
  9. 卸载该插件
  10. 导航到来源选项卡
  11. 向下滑动刷新你的软件源
  12. 导航到搜索选项卡
  13. 在搜索栏中输入 libiosexec1
  14. 安装 Hayden Seay 的 libiosexec1
  15. 再次尝试安装导致错误的插件

TIP

2021 年 5 月，Procursus 尝试迁移到某个版本的 dpkg，该版本不再处理此安装错误，因为开发者本应在脚本中包含 #!/bin/sh 或 #!/bin/bash 调用。在撰写本文时，Procursus 的首席开发者创建了一个补丁 libiosexec，重新添加了此错误处理。如果你仍然遇到此错误，可能是你有一段时间没有更新了。

# nitoTV 问题

# 应用无法启动

  1. 如果点击 nitoTV 应用后无法打开，可能是因为设备处于未越狱状态。这种情况通常发生在 Apple TV 重启或关机后重新开机时。
  2. 重新执行越狱将使 nitoTV 恢复全部功能。这可以通过 checkra1n 或启动其他越狱工具并点击Jailbreak来完成。

# 无法安装/下载/搜索更新

  1. 打开 nitoTV，导航到顶部栏的来源选项卡，高亮选择 nitoTV 软件源。
  2. 点击并按住触控板按钮约一秒钟，然后松开，系统将提示移除该软件源。
  3. 删除后，点击添加，输入以下软件源 URL：[https://nitosoft.com/checkra1n/]，然后点击确定。
  4. 稍等片刻，该软件源应会显示在列表中。
  5. 完成后，调出应用多任务切换界面，向上滑动 nitoTV 以终止该应用，然后从主屏幕重新启动 nitoTV。

# nitoTV 在添加新软件源时崩溃

  1. 打开 nitoTV，导航到顶部栏的已安装选项卡，高亮 nitoTV。
  2. 点击应用名称下方的修改，选择降级，然后选择最适合的版本。
  3. 另一种可能的修复方法是通过 SSH 登录。
  4. 找到你的 Apple TV 设备的 IP 地址，使用命令 ssh root@Apple TV IP 地址，然后使用 alpine 作为密码登录。
  5. 登录后，输入以下命令来添加你选择的软件源：echo deb URL_ADDRESS > /etc/apt/sources.list.d/sources.list

# 重新加载桌面时的问题

  1. 在某些情况下，需要从多任务窗口终止 nitoTV。这可以通过在 Siri Remote 上按两次 TV 按钮来完成。

# 应用不显示

  1. 简单的重新加载桌面可以解决这些奇怪的情况。
  2. 仔细检查 nitoTV 顶部栏的控制台日志，选择设置。
  3. 然后点击查看日志
